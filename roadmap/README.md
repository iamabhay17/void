# Design Engineer Learning Roadmap

A comprehensive curriculum for becoming a high-level Design Engineer — someone who deeply understands design principles, motion design, interaction design, and how to implement them in code using React.

## Goal

Build polished, intentional, and tasteful interfaces with high-quality animations and interactions like those seen in products from Linear, Stripe, and Vercel.

---

## Roadmap Structure

### Phase 1: Foundations
1. **[Design Foundations](./01-design-foundations/)** — Visual hierarchy, layout systems, typography, color, spacing
2. **[Interaction Design](./02-interaction-design/)** — Feedback, micro-interactions, gesture patterns

### Phase 2: Motion & Animation Theory
3. **[Motion Design](./03-motion-design/)** — Easing, timing, choreography, motion principles
4. **[Browser Rendering Pipeline](./04-browser-rendering-pipeline/)** — Layout, paint, composite, GPU acceleration

### Phase 3: React Architecture for UI
5. **[React UI Architecture](./05-react-ui-architecture/)** — Component composition, patterns, state management
6. **[React Hooks for UI & Animation](./06-react-hooks-ui-animation/)** — Measurement, lifecycle, performance hooks

### Phase 4: Animation Implementation
7. **[React Animation Patterns](./07-react-animation-patterns/)** — State-driven animations, transitions, orchestration
8. **[Framer Motion Deep Dive](./08-framer-motion/)** — Variants, layout animations, gestures, scroll

### Phase 5: Advanced Techniques
9. **[SVG & Vector Animations](./09-svg-animation/)** — Path animations, morphing, icon systems
10. **[Advanced UI Engineering](./10-advanced-ui-engineering/)** — Physics, gestures, canvas, state machines

### Phase 6: Mastery
11. **[UI Polish & Taste](./11-ui-polish-taste/)** — Subtle polish, restraint, consistency, motion language

---

## How to Use This Roadmap

1. **Study each topic in order** — The curriculum builds progressively
2. **Read the guidelines.md** — Each contains theory, implementation, and exercises
3. **Complete the practice exercises** — Understanding comes from doing
4. **Build real projects** — Apply concepts to your own work
5. **Revisit advanced topics** — Some concepts need multiple passes
